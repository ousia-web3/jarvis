# 간단 시작 요청

사용자는 복잡한 Human Brief 양식을 먼저 작성하지 않습니다. 아래처럼 짧게 요청하면 에이전트 팀이 Human Brief를 자동 생성한 뒤 PRD, TASK, 실행 계획, 리스크 검토까지 이어갑니다.

## IDE 창에 넣는 최소 요청

```text
docs 기준으로 에이전트 팀 모드로 진행해줘.
요청: [하고 싶은 일 한 줄]
```

## 더 짧은 요청

```text
[하고 싶은 일] 해줘. docs의 에이전트 팀 운영 규칙대로 자동 진행해.
```

## 예시

```text
유튜브 쇼핑몰 MVP 기획하고 랜딩 페이지 작업까지 진행해줘.
docs의 에이전트 팀 운영 규칙대로 Human Brief부터 자동 생성해서 진행해.
```

```text
신규 SaaS 아이디어를 PRD/TASK로 정리하고 MVP 개발 순서까지 만들어줘.
Human Brief는 먼저 초안으로 자동 작성해줘.
```

## 자동 처리 규칙

1. 신규 작업 요청이면 먼저 요청 슬러그를 정한다.
2. 요청 슬러그를 정한 직후 `scripts/start-jarvis-request.ps1`를 실행해 대시보드 서버와 첫 이벤트 로그를 준비한다.
3. 스크립트가 반환한 URL은 OS 기본 브라우저가 아니라 현재 AI툴 브라우저 또는 프리뷰 표면에서 연다.
4. `work-requests/YYYY-MM-DD-request-slug/` 폴더를 만들고 관련 자료 보관 위치를 정한다.
5. 사용자 원문 요청을 Human Brief 초안으로 변환한다.
6. Jarvis가 전략, 우선순위, 성공 기준을 정리한다.
7. Friday가 Owner(To), CC, 산출물, 완료 기준을 태스크로 분해한다.
8. EVE, Joi, TARS, C3PO가 필요한 실무를 맡는다.
9. Data, KITT/TRON, Diagnostic Agent가 분석과 리스크 쉴드를 수행한다.
10. 승인, 배포, 삭제, 외부 전송, 비밀키, 개인정보, 법무 판단처럼 위험한 항목만 인간 대표에게 확인한다.

완료된 대화창이나 완료된 작업 요청에서 이어지는 추가 작업이면 기존 `requestId`를 재사용한다. 같은 `requestId`로 `scripts/start-jarvis-request.ps1`를 다시 실행해 상태를 `In Progress`로 바꾸고 대시보드/Virtual Office 시각화를 다시 보여준다.

## 되묻지 않고 진행해도 되는 항목

- 문서 구조 정리
- PRD/TASK 초안 작성
- 내부용 기획안 작성
- 파일 생성과 수정
- 코드 리딩과 비파괴적 검증
- 합리적인 기본값 선택

## 반드시 확인해야 하는 항목

- 기존 파일 대량 삭제 또는 되돌리기
- 외부 배포, 외부 전송, 공개 저장소 업로드
- 결제, 개인정보, 비밀키, 법무 리스크
- 프로젝트 방향을 크게 바꾸는 전략 결정
