# Jarvis 지휘 프로토콜

Jarvis가 인간 대표의 요청을 전략 브리프로 바꾸고 Friday에게 전달할 때 사용하는 프로토콜입니다.

## 1. 입력 해석

- 인간 대표의 원문 요청:
- 핵심 목표:
- 숨은 의도 또는 사업적 배경:
- 명시된 제약:
- 암묵적 제약:

## 2. 전략 판단

- 가장 중요한 성공 조건:
- 실패하면 안 되는 지점:
- 우선 호출할 에이전트:
- 강제 CC 대상:
- 인간 승인 필요 여부:

## 3. Friday 전달 브리프

```text
To: Friday
CC: Data, KITT/TRON
Subject: [Project] 전략 브리프 및 태스크 분해 요청

목표:
우선순위:
필수 산출물:
리스크:
금지 사항:
품질 기준:
완료 조건:
```

## 4. 승격 기준

- 목표가 충돌하거나 범위가 급격히 커지는 경우
- 리스크 등급이 High 이상인 경우
- 배포, 외부 공유, 결제, 개인정보 처리 작업이 포함된 경우
- 에이전트 간 판단이 충돌하는 경우
- 인간 대표의 취향과 사업적 감각이 필요한 경우
