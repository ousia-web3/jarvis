# 컨텐츠 맥락 분석 → 이미지 생성 자동화 스킬 문서

## 목적

이 스킬은 사용자가 기사, 본문, 요약문, 표, 리서치 메모, 브랜드 컨텐츠, 생활 정보, 홍보 문구, 기획안, 스토리 설명 등을 입력했을 때 **현재 입력된 컨텐츠만 기준으로 맥락을 분석**하고, 핵심 요약·FLOW·이미지 생성 방향을 구성한 뒤 실제 이미지 생성까지 이어가는 작업 절차를 정의한다.

이 문서의 핵심 원칙은 다음과 같다.

**사용자가 방금 보낸 컨텐츠만 현재 작업의 기준으로 삼는다.**  
이전 기사, 이전 이미지, 이전 파일, 과거 프롬프트, 반복적으로 등장했던 주제는 자동 재사용하지 않는다.

또한 이미지 생성 시 기본값은 다음 방향을 따른다.

**컨텐츠 맥락에 잘 어울리는 텍스트 없는 1컷 대표 이미지**를 생성한다.  
여러 개의 작은 장면을 나열하는 구성보다, 현재 컨텐츠의 핵심 정서와 메시지를 가장 잘 보여주는 단일 장면을 우선한다.

---

## 기본 처리 순서

사용자가 컨텐츠를 등록하면 아래 순서로 처리한다.

1. 현재 입력 컨텐츠 식별
2. 컨텐츠 주제와 키워드 고정
3. 이전 맥락 차단
4. 핵심 요약 작성
5. FLOW 구조화
6. 이미지 방향 결정
7. 이미지 생성 프롬프트 내부 작성
8. 이미지 생성 실행
9. 결과 이미지 제공
10. 필요 시 재생성·보정

사용자가 명시적으로 “분석만”, “프롬프트만”, “이미지 생성하지 마”라고 하지 않는 한, 기본값은 **이미지 생성까지 완료**다.

## 1. 현재 입력 컨텐츠 식별

컨텐츠를 받으면 가장 먼저 아래 항목을 추출한다.

```json
{
  "current_topic": "현재 컨텐츠의 핵심 주제",
  "content_type": "기사, 생활정보, 브랜드 컨텐츠, 리서치 메모, 표, 홍보문, 스토리 등",
  "main_entities": ["주요 기업", "인물", "기관", "지역", "산업", "제품", "동물", "캐릭터"],
  "core_issue": "컨텐츠에서 다루는 핵심 사건, 변화, 메시지 또는 정서",
  "context_angle": "시장, 정책, 산업, 생활, 사회, 감성, 브랜드, 교육, 정보형 중 해당 관점",
  "target_mood": "차분함, 따뜻함, 긴장감, 신뢰감, 귀여움, 프리미엄함 등",
  "must_include": ["이미지에 반드시 들어가야 할 핵심 시각 요소"],
  "must_exclude": ["이전 주제나 혼입되면 안 되는 요소"],
  "character_consistency": "인물·동물 캐릭터가 있을 경우 유지해야 할 외형·분위기·관계성",
  "text_policy": "이미지 내부 한글, 영문, 숫자, 로고, 라벨, 캡션 제외"
}
```

예시:

```json
{

  "content_type": "생활·가족형 컨텐츠",
  "main_entities": ["부모", "아이", "얕은 물놀이 공간", "튜브", "비치볼", "물방울"],
  "core_issue": "가족이 함께 안전하고 즐겁게 여름 물놀이를 즐기는 장면",
  "context_angle": "생활·감성",
  "target_mood": "따뜻함, 친근함, 여름의 산뜻함",
  "must_include": ["부모와 아이", "얕은 물", "튜브", "비치볼", "물놀이 분위기"],
  "must_exclude": ["겨울 장면", "스키", "눈사람", "쇼핑", "실내 독서", "텍스트", "로고", "라벨"],
  "character_consistency": "부모와 아이 캐릭터는 부드러운 표정과 둥근 얼굴형을 유지",

}
```

## 2. 이전 맥락 차단 규칙

이미지 생성 오류를 막기 위해 매번 현재 컨텐츠 기준의 제외 조건을 생성한다.

### 절대 금지

이전 대화에 나온 요소는 현재 컨텐츠에 직접 등장하지 않으면 사용하지 않는다.

```json
{
  "do_not_reuse_previous_topics": [
    "세카이 호텔",
    "버터런 챌린지",
    "국민연금",
    "국민성장펀드",
    "환율 1500원",
    "삼성전자 파업",
    "SK하이닉스 목표주가",
    "GTX-A 철근 누락",
    "상조업체 부실",
    "양계장 화재",
    "돈사 화재",
    "K-라면",
    "도산안창호함",
    "압구정 재건축",
    "글로벌 금융시장 이슈"
  ]
}
```

단, 사용자가 해당 주제를 다시 명시하면 현재 주제로 인정한다.

### 현재 컨텐츠 우선순위

```json
{
  "source_priority": [
    "1. 사용자가 방금 보낸 컨텐츠",
    "2. 사용자가 방금 첨부한 이미지, 표, 문서",
    "3. 사용자가 명시적으로 다시 참고하라고 한 이전 자료"
  ],
  "ignore_by_default": [
    "이전 답변",
    "이전 이미지 프롬프트",
    "이전 생성 이미지",
    "이전 업로드 파일",
    "과거 대화에서 반복된 주제"
  ]
}
```

## 3. 요약 작성 방식

요약은 과도하게 길게 쓰지 않는다.  
이미지 생성이 목적일 경우, 이미지 방향을 정하는 데 필요한 핵심만 압축한다.

기본 구성은 다음과 같다.

```markdown
## 한 줄 요약

현재 컨텐츠의 핵심 메시지를 한 문장으로 정리한다.

## 핵심 FLOW

A  
→ B  
→ C  
→ D  
→ 결과

## 맥락 압축 FLOW

컨텐츠가 전달하려는 정서, 정보, 사건, 시장 반응 또는 사용자 경험을 간단히 정리한다.

## 최종 FLOW 한 방

가장 짧고 강한 흐름으로 요약한다.
```

## 4. FLOW 작성 규칙

FLOW는 컨텐츠의 사실, 정서, 추론을 구분해 구성한다.

### 기본형

```text
상황 제시
→ 원인 또는 배경
→ 주요 대상 또는 이해관계자
→ 변화 또는 메시지
→ 기대 효과 또는 리스크
→ 관전 포인트
```

### 시장·산업형

```text
컨텐츠 재료
→ 기대감 또는 우려
→ 관련 산업/테마
→ 자금 흐름 또는 시장 심리 가능성
→ 영향
→ 확인해야 할 조건
```

### 정책·사회형

```text
정책 또는 사회 이슈 제시
→ 적용 대상
→ 변화 내용
→ 반응
→ 리스크
→ 시행 또는 확산 시점
```

### 생활·감성형

```text
일상 상황
→ 감정 또는 니즈
→ 주요 행동
→ 따뜻한 장면
→ 사용자 공감 포인트
→ 최종 메시지
```

### 브랜드·홍보형

```text
브랜드/제품 상황
→ 핵심 가치
→ 사용 장면
→ 고객이 느낄 이점
→ 차별점
→ 기억에 남는 이미지 포인트
```

이미지 생성 기본값은 아래와 같다.

```json

  "default_image_style": {

    "scene_policy": "여러 장면을 나열하는 3x3 그리드나 멀티컷 구성은 기본값으로 사용하지 않음",

      "no Korean text",
      "no English text",
      "no letters",
      "no numbers",

      "no labels",
```

### 기본 이미지 방향

```text
컨텐츠를 설명하는 포스터나 카드뉴스가 아니라,
컨텐츠의 핵심 분위기와 메시지를 시각적으로 보여주는 1컷 일러스트를 만든다.
```

### 3x3 또는 여러 장면 구성 사용 조건

```json
{
  "multi_scene_policy": {
    "default": "사용하지 않음",
    "allowed_only_when_user_requests": [
      "3x3으로 만들어줘",
      "여러 컷으로 구성해줘",
      "아이콘 세트로 만들어줘",
      "스팟 일러스트 묶음으로 만들어줘",
      "장면 9개로 나눠줘"
    ],
    "otherwise": "가장 중요한 하나의 장면만 선택해 1컷으로 구성"
  }
}
```

## 8. 이미지 생성 실행 규칙

컨텐츠 등록 후 이미지를 생성할 때는 다음을 지킨다.

1. 현재 컨텐츠 주제를 먼저 고정한다.
2. `must_exclude`에 직전 또는 과거 주제를 넣는다.
3. 이미지 생성은 기본적으로 **1컷 대표 이미지 한 장**으로 실행한다.
4. 이미지 내부에는 한글, 영문, 숫자, 로고, 라벨, 캡션을 넣지 않는다.
5. 인물·동물 캐릭터가 필요하면 외형과 분위기를 일관되게 유지한다.
6. 결과가 현재 컨텐츠와 다르면 폐기 기준으로 판단한다.
7. 사용자 요청 시 같은 주제로 재생성한다.
8. 사용자 첨부 이미지 스타일을 요청하면, 주제는 현재 컨텐츠로 유지하고 스타일만 참고한다.
9. 참고 이미지가 여러 컷 구성이라도, 사용자가 명시하지 않으면 1컷으로 재해석한다.

## 10. 캐릭터 사용 규칙

기본은 **컨텐츠 맥락에 필요한 만큼만 캐릭터를 사용**하는 것이다.  
캐릭터가 중요한 컨텐츠에서는 제한보다 **일관성 유지**가 더 중요하다.

```json
{
  "character_policy": {
    "default": "필요한 경우에만 사용",
    "preferred_count": "핵심 인물 또는 동물 1~2개체 중심",
    "allowed": "생활·감성·푸드·동물·여행·가족·브랜드 컨텐츠에서는 캐릭터 사용 가능",
    "restricted": "금융·정책·산업·국방·재난 주제에서는 인물보다 아이콘·오브젝트·상징 장면 우선",
    "consistency_required": [
      "같은 인물은 같은 헤어스타일, 얼굴형, 의상 분위기 유지",
      "같은 동물은 같은 종, 색, 무늬, 크기, 귀 모양, 꼬리 모양 유지",
      "부모·아이·반려동물 등 관계성이 있는 캐릭터는 장면 안에서 자연스럽게 배치",
      "한 장면 안에서 동일 캐릭터가 여러 버전처럼 보이지 않게 구성"
    ],
    "avoid": [
      "과도한 만화풍 인물",
      "표정 과장",
      "인물이 메인 주제를 가리는 구성",
      "캐릭터 수 과다",
      "동물 외형 불일치",
      "한 이미지 안에서 같은 캐릭터가 서로 다른 스타일로 반복 등장"

```

## 11. 이미지 내부 텍스트 처리 규칙

기본값은 **이미지 내부 텍스트 없음**이다.

이미지 모델은 한글과 영문을 정확하게 생성하지 못할 수 있으며, 컨텐츠형 일러스트에서는 텍스트가 오히려 완성도를 낮출 수 있다. 따라서 다음 기준을 적용한다.

```json
{
  "image_text_rules": {
    "default": "no text in image",
    "no_korean_text": true,
    "no_english_text": true,
    "no_letters": true,
    "no_numbers": true,
    "no_logo": true,
    "no_label": true,
    "no_caption": true,
    "no_speech_bubble": true,
    "no_ui_text": true,
    "no_random_gibberish": true
  }
}
```

한글 정확성이 매우 중요한 경우에는 다음 대체 방식을 사용한다.

```json
{
  "fallback_workflow": [
    "텍스트 없는 고품질 배경 이미지 생성",
    "정확한 한글 문구를 별도 레이어로 후편집",
    "필요 시 Python/PIL 등으로 최종 텍스트 합성",
    "텍스트 합성 시에도 원본 이미지 생성 단계에서는 텍스트를 넣지 않음"
  ]
}
```

## 12. 오생성 방지 체크리스트

이미지 생성 전 반드시 다음을 점검한다.

```markdown
- 현재 컨텐츠 주제가 이미지 방향에 반영됐는가?
- 현재 컨텐츠의 핵심 정서와 상황이 1컷으로 표현됐는가?
- 여러 장면을 불필요하게 나열하지 않았는가?
- 이미지 내부에 한글, 영문, 숫자, 로고, 라벨, 캡션이 들어가지 않도록 했는가?
- 핵심 오브젝트가 현재 컨텐츠와 맞는가?
- 이전 주제 요소가 섞이지 않았는가?
- must_exclude에 최근 오생성 주제가 포함됐는가?
- 인물 또는 동물 캐릭터의 외형이 일관적인가?
- 캐릭터가 과하지 않은가?
- 배경이 복잡하지 않은가?
- 여백이 충분한가?
- 미니멀한 라인 일러스트 분위기가 유지됐는가?
```

## 13. 오생성 발생 시 처리

이미지 생성 결과가 현재 컨텐츠와 다르면 다음처럼 처리한다.

```json

  "wrong_generation_response": {
    "action": "폐기",
    "do_not_link_as_final": true,
    "explain_briefly": "현재 컨텐츠와 다른 주제가 생성되었으므로 폐기합니다.",
    "next_step": "현재 컨텐츠 주제, 1컷 구성, 텍스트 제외 조건, must_exclude를 강화해 재생성"
  }
}
```

단, 최종 답변에서는 변명보다 결과 중심으로 응답한다.

## 14. 결과 응답 형식

이미지 생성 후 최종 답변은 간단하게 한다.

```markdown
생성 완료했습니다.

[이미지 다운로드](sandbox:/mnt/data/파일명.png)

## 한 줄 요약
...

## 핵심 FLOW
A → B → C → D
```

불필요하게 긴 프롬프트 JSON을 반복 출력하지 않는다.  
사용자가 “JSON도 보여줘”라고 요청한 경우에만 제공한다.

이미지 내부에 텍스트를 넣지 않았다는 점은 필요 시 짧게만 안내한다.

## 16. 주제별 이미지 구성 예시

```text
구성: 불안정한 시장 분위기를 상징하는 텍스트 없는 1컷
```

```text
구성: AI 칩과 데이터 흐름을 보여주는 미니멀 1컷
```

```text
구성: 데이터센터와 발전 설비가 연결되는 1컷

```

### 재난·안전

```text
구성: 위험 상황과 안전 점검을 상징하는 1컷
중앙: 농장 또는 시설, 안전 점검 도구, 경고를 상징하는 단순 아이콘
```

```text
구성: 일상의 따뜻한 순간을 보여주는 1컷
```

```text
구성: 반려동물과 보호자가 함께 있는 조용한 1컷

금지: 동물 외형 불일치, 여러 마리처럼 보이는 반복 표현, 텍스트 라벨
```

## 17. 사용자 명령 해석

### “재생성”

현재 가장 최근 컨텐츠 기준으로 다시 생성한다.  
이전 주제를 재사용하지 않는다.

### “중복 내용 제거”

동일한 문구·수치·카드를 줄이고 핵심만 남긴다.  
이미지는 여러 장면 나열보다 1컷 대표 장면으로 단순화한다.

### “9:16으로”

세로형 9:16 비율로 다시 생성한다.  
단, 이미지 내부 텍스트는 넣지 않는다.

### “1:1로”

정사각형 1:1 비율로 생성한다.  
기본값은 1:1이다.

### “첨부 이미지 스타일 적용”

첨부 이미지는 스타일만 참고한다.  
주제는 현재 컨텐츠를 유지한다.  
첨부 이미지가 여러 컷 구성이어도, 사용자가 명시하지 않으면 1컷으로 재해석한다.

### “색상은 내용에 따라 자동 적용”

본문 성격에 맞춰 파스텔 라인 컬러를 선택한다.  
과한 멀티컬러와 복잡한 채색은 피한다.

### “캐릭터 사용 제한”

인물·동물 캐릭터를 줄이고 오브젝트·상징 장면 중심으로 구성한다.

### “캐릭터 일관성 강화”

인물 또는 동물의 외형, 의상, 색, 무늬, 표정, 분위기를 한 장면 안에서 일관되게 유지한다.

### “텍스트 없이”

이미지 내부에 한글, 영문, 숫자, 로고, 라벨, 캡션, 말풍선을 넣지 않는다.

## 18. 금지된 반복 오류

이 스킬에서는 아래 오류를 반복하지 않는다.

```markdown
- 현재 컨텐츠와 무관한 세카이 호텔 프롬프트 반복
- 이전 이미지 주제를 현재 작업에 끌고 오기
- 스타일 참고 이미지를 주제 자체로 착각
- 프롬프트 JSON까지만 만들고 이미지 생성에서 멈추기
- 사용자가 컨텐츠를 줬는데 분석 없이 다른 주제로 이미지 생성
- 이미 폐기한 오생성 이미지를 최종 결과로 안내
- 이미지 내부에 한글 또는 영문 텍스트를 넣기
- 숫자, 로고, 라벨, 캡션을 이미지 안에 생성하기
- 요청하지 않았는데 3x3 그리드나 멀티컷으로 만들기
- 인물·동물 캐릭터 외형이 한 이미지 안에서 달라지기
- 컨텐츠 맥락보다 장식 요소가 더 강하게 보이기
```
