## 3. 주요 내용 · 개선 사항

### 3-1. Cat-first V5 아트 시스템

각 기존 이미지를 단순 스타일 참고가 아닌 **배경 거리 레퍼런스**로 사용한다. 카메라 위치, 배경 사물의 크기, 바닥 원근, 주요 여백은 유지하되 고양이 캐릭터와 선화·채색을 V5 기준으로 교체한다.

**상세 내용**

- 초기 2000년대 일본 손그림 극장판 고양이 판타지의 깨끗하고 가벼운 시각 언어를 사용하되 기존 캐릭터·프레임·프로덕션 디자인은 복제하지 않는다.
- 얇은 웜 차콜 선, 넓은 무광 셀 색면, 단일 그림자, 불투명하게 칠한 배경을 사용한다.
- 수채화, 동화책 워시, 종이결, 파스텔 초크, 3D, CGI, 실사, 과도한 털 묘사를 금지한다.
- 임의의 광각 확장, 긴 복도, 과장된 소실점, 불필요한 전경 프레이밍을 추가하지 않는다.
- 사람은 필요한 장면에만 손·옆모습·등·무릎 정도로 단순화하고 고양이보다 먼저 읽히지 않게 한다.

### 3-2. 86개 자산 생성과 캐릭터 다양성

14 Cat ID를 시나리오·개념·챕터·홈에 분산하고 단일 캐릭터가 전체 자산의 12%를 넘지 않게 한다. 품종은 행동 성격의 원인처럼 서술하지 않고 시각 다양성 메타로만 사용한다.

**상세 내용**

- 홈 6, 시나리오 11, 주요 개념 27, 챕터 42를 각기 고유 이미지로 유지한다.
- 다묘 장면은 2~3 Cat ID를 사용하되 같은 조합의 연속 반복을 피한다.
- 건강·안전 장면은 웃는 아픈 고양이, 구토물·배설물 묘사, 치료 도구, 공격·공포 과장을 금지한다.
- 각 생성본은 원본 PNG와 1024×1024 WebP를 분리 저장하고, public에는 최적화본만 추가한다.

### 3-3. 콘텐츠 이미지 manifest 전환

이미지 경로와 대체 텍스트만 가진 기존 하드코딩 객체를 86개 통합 manifest로 전환한다. 런타임 manifest는 콘텐츠·Cat ID·버전·프롬프트 참조를 추적하고, 상세 품종·모색·나이 정보와 장면 배정은 `12-cat-first-v5-cat-cast-content-map.md`를 SSOT로 둔다.

**상세 내용**

- 86개 `id`와 `slug`는 unique여야 한다.
- `label`, 한글 `alt`, `catIds`, `kind`, `version`, `promptRef`, `src`를 필수로 둔다.
- 품종·모색·나이·체형 고정값은 같은 파일의 `CAT_CAST`와 `docs/12`에서 추적한다.
- 기존 `getScenarioIllustration`, `getConceptIllustration` API는 새 manifest를 읽는 호환 어댑터로 유지한다.
- V2 경로에 문제가 있으면 기존 V1 resolver로 롤백 가능한 구조를 유지한다.

### 3-4. 목록 간결화와 상세 이미지 집중

홈의 많이 묻는 궁금증, 탐색의 상황별 전체보기, 상세 하단의 관련 항목은 제목·분류·설명 중심 목록으로 유지한다. V5 이미지는 사용자가 목록 카드를 선택한 뒤 진입하는 시나리오·개념·챕터 상세 상단에서만 노출한다.

**상세 내용**

- 목록 카드는 분류 배지 → 제목 → 설명/CTA 순서를 유지하고 이미지 및 `AI 생성` 배지를 렌더링하지 않는다.
- 카드 링크의 키보드·focus ring·광고 게이트 동작을 보존한다.
- 상세 대표 이미지는 1:1 원본과 정사각형 슬롯을 유지하고 PC/MO 브라우저 검수로 눈·귀·발 잘림을 확인한다.
- 상세 이미지 로드 실패와 무관하게 본문·관찰 항목·관련 목록은 사용할 수 있어야 한다.

### 3-5. 문서화와 품질 게이트

최종 프롬프트, 캐릭터 바이블, 콘텐츠 배정, 생성 일자, 경로, 검증 결과를 프로젝트 `docs`와 work-request 증거 폴더에 남긴다.

**상세 내용**

- 생성본 86개 missing 0, 배포본 unique hash 86/86을 목표로 한다.
- WebP 권장 180KB 이하, 예외 250KB 이하를 적용한다.
- 한글 `alt` 100%, 프롬프트 참조 100%, Cat ID 메타 100%를 자동 검사한다.
- 외부 배포는 별도 사용자 요청 전까지 차단한다.

## 4. 주요 정책 또는 흐름도

### 4-1. 정책 정의

| 항목 | 값 / 규칙 | 비고 |
| --- | --- | --- |
| 자산 기준 | 콘텐츠 80 + 홈 6 = 86개 | 모든 자산 고유 이미지 |

### 4-2. 실행 흐름

1. 기존 PNG를 장면별 배경 거리 레퍼런스로 읽는다.
2. `12-cat-first-v5-cat-cast-content-map.md`에서 Cat ID와 장면 초점을 선택한다.
3. `11-cat-first-v5-image-prompt-guide.md` 공통 프롬프트와 카테고리 가드레일을 결합한다.
4. built-in `image_gen`으로 콘텐츠마다 1회씩 생성하고 원본 PNG를 보관한다.
5. 1024×1024 WebP로 최적화하고 크기·해시·규격을 검사한다.
6. manifest 행을 상세 대표 이미지에 연결하고 홈·탐색·관련 목록은 텍스트 카드로 제공한다.
7. 접근성·안전·다양성·성능·화면 회귀를 검증한다.
8. 로컬 완료 상태로 종료하고 GitHub/Vercel은 후속 승인까지 대기한다.

### 4-3. 전환 흐름


## 5. 특이사항

- 특정 영화의 캐릭터·프레임을 복제하지 않고 시대·매체 특성 수준의 일반 시각 언어만 사용한다.
- 이미지 생성은 확률적이므로 신체 오류, 고양이 모색 드리프트, 안전 장면 과장을 접촉 시 수정 대상으로 기록한다.
- 기존 이미지와 V3/V4/V5 홈 샘플은 롤백·비교 증거로 보존한다.
- 기존 IA와 URL 구조는 확정 상태로 재사용하며 신규 route·데이터 모델·외부 API는 추가하지 않는다.
- GitHub와 Vercel 배포는 이번 완료 기준에 포함하지 않는다.

## 2026-07-16 스타일 일관성 수락 게이트

Cat-first V5 이미지는 단순히 2D 애니메이션풍이면 충분하지 않다. 최종 수락 기준은 `public/images/v2/scenarios/grooming.webp`와 같은 계열의 선, 채색, 배경 거리, 고양이 비중을 유지하는 것이다.

수락 조건:

- 고양이가 첫 시선 대상이어야 한다.
- 배경은 생활 맥락을 설명하되 고양이를 압도하지 않아야 한다.
- 기존 생성 이미지와 같은 중거리 배경 원근을 유지해야 한다.
- 따뜻한 실내광, 세이지/더스티 블루 보조색, 자연스러운 셀 채색이 유지되어야 한다.
- 수채화, 동화책, 종이 질감, 베이지 단색 워시, 3D/CGI, 사진풍, 두꺼운 벡터선은 불합격이다.
- 건강/안전 장면은 비진단 관찰 이미지여야 하며 경고 아이콘, 의료 상징, 공포 연출을 쓰지 않는다.

이번 보정은 위 게이트에 따라 21개 이미지를 교체했으며, 증거는 `artifacts/images/cat-first-v2/qa/style-anchor-correction-2026-07-16/`와 `work-requests/2026-07-09-nyangtology-content-review/evidence/cat-first-v5-style-anchor-correction-verification.md`에 둔다.
