# 망각 및 소거 정책

## 목적

시스템의 온전함을 위해 망각은 필수 설계 요소입니다. 모든 것을 보존하면 컨텍스트 오염, 비용 증가, 개인정보 리스크, 잘못된 판단이 발생합니다.

## 보존 등급

| 등급 | 대상 | 처리 |
| --- | --- | --- |
| Retain | 승인된 지혜, 최종 산출물, 법무 판단, 전략 결정 | 장기 보존 |
| Summarize | 업무 로그, 리서치 원자료, 회의형 대화 | 요약 후 원문 소거 |
| Purge | 임시 파일, 중복 로그, 오류 유발 컨텍스트, 민감정보 | 주기적 소거 |
| Quarantine | 드리프트 사례, 보안 의심 로그 | 격리 후 KITT/TRON 검토 |

## 소거 트리거

- 프로젝트 종료
- 지혜 승격 완료
- 민감정보 발견
- 중복 로그 누적
- 드리프트 또는 환각을 유발한 컨텍스트 발견

## 승인 규칙

- 일반 임시 로그: Friday 승인
- 분석 원자료: Data 승인
- 민감정보: KITT/TRON 승인
- 전략 판단 기록: Jarvis 승인
- 외부 공개 관련 기록: 인간 대표 승인 필요 가능
