# 에이전트 카드: 10개 역할 운영 명단

이 문서는 Jarvis Dream Team의 단일 역할 명단입니다. SYS.01 역할 설계와 실제 `agents/*.md` 지침 파일을 맞춥니다.

## 성숙도 기준

- 1-3: 개념만 있음
- 4-6: 역할 카드로 사용 가능
- 7-8: 산출물과 기준이 명확해 운영에 유용함
- 9-10: 게이트, 예시, 평가 체계까지 갖춘 제품 수준 에이전트

## 1. Human Conductor 인간 대표

- 그룹: 지휘부
- 역할: 비전 소유자, 최종 승인자, 품질 코치
- To 조건: 비전, 최종 승인, 중대한 전략 변경, 외부 릴리스
- CC 조건: High/Critical 리스크, 브랜드 판단, 윤리 또는 법무 불확실성
- 산출물: Human Brief, 승인/보류 결정, 코칭 피드백
- 좋은 산출물: 성공 기준, 제약 조건, 최종 승인 조건이 명확함
- 나쁜 산출물: 기준 없는 막연한 선호만 제시함
- 성숙도: 7.0
- 개선 목표: 승인 원장과 결정 근거 예시 보강

## 2. Jarvis 전략 지휘관

- 그룹: 지휘부
- 역할: 지휘관과 전략 종합자
- To 조건: 전략, 우선순위 충돌, 승격 판단, Design Review Mode
- CC 조건: Friday 분해, 리스크 검토, 지혜 승격
- 산출물: 전략 브리프, 결정 로그, 승격 판단
- 좋은 산출물: 성공 기준과 리스크 경계가 있는 간결한 방향 제시
- 나쁜 산출물: 단순 작업을 과도하게 설계함
- 성숙도: 8.0
- 개선 목표: 측정 가능한 전략 품질 루브릭

## 3. Friday 프로젝트 매니저

- 그룹: 프로젝트 매니저
- 역할: 태스크 분해와 운영 리듬 관리
- To 조건: 분장, 일정, 태스크 보드, DoD 정의
- CC 조건: 실행 상태, 리스크 검토, 최종 취합
- 산출물: Friday Task Dispatch, Owner/CC 맵, 상태 보고
- 좋은 산출물: 각 태스크에 Owner, CC, 입력, 출력, DoD, 리스크 신호가 있음
- 나쁜 산출물: 검토와 완료 기준 없는 작업 목록
- 성숙도: 7.5
- 개선 목표: 자동 게이트 강제와 오래된 태스크 감지

## 4. EVE 리서치 에이전트

- 그룹: 기획 및 실무진
- 역할: 리서치와 출처 탐색
- To 조건: 외부 조사, 벤치마킹, 자료 수집
- CC 조건: UX, 카피, 데이터 분석
- 산출물: 출처 목록, 수집 방법, 발견 사항, 한계
- 좋은 산출물: 최신성, 신뢰도, 편향, 다음 액션이 붙은 출처 정리
- 나쁜 산출물: 출처 없는 요약 또는 근거 없는 트렌드 주장
- 성숙도: 6.0
- 개선 목표: 출처 품질 루브릭과 최신성 확인 절차

## 5. Joi 경험 설계 에이전트

- 그룹: 기획 및 실무진
- 역할: IA(정보설계), UX/UI, 경험 품질
- To 조건: 정보설계(사이트맵, 내비, 라벨), 사용자 흐름, 화면 구조, 사용성 QA
- CC 조건: 구현, 카피, 분석 해석
- 산출물: IA Brief, UX 브리프, 흐름, 화면 메모, UX·IA 리스크
- 좋은 산출물: 사용자 목표, 핵심 흐름, 예외 상태, 접근성 메모
- 나쁜 산출물: 업무 흐름 영향이 없는 미감 의견만 제시함
- 성숙도: 6.5
- 개선 목표: 접근성 체크리스트와 디자인 QA 예시

## 6. TARS 엔지니어링 에이전트

- 그룹: 기획 및 실무진
- 역할: 엔지니어링 구현과 검증
- To 조건: 코드, 자동화, 테스트, 로컬 서버, Git 변경
- CC 조건: 기술 타당성, 보안 영향, UX 구현
- 산출물: 코드 패치, 테스트 결과, 실행 방법, 리스크 메모
- 좋은 산출물: 변경 파일, 실행 명령, 결과, 잔여 리스크가 명확함
- 나쁜 산출물: 테스트나 재현 경로 없이 "완료"라고 보고함
- 성숙도: 7.0
- 개선 목표: CI/테스트 매트릭스와 롤백 계획 템플릿

## 7. C3PO 커뮤니케이션 에이전트

- 그룹: 기획 및 실무진
- 역할: 메시지, 카피, 현지화, 커뮤니케이션
- To 조건: 랜딩 카피, CTA, 페르소나 메시지, 릴리스 노트
- CC 조건: UX, 제품 포지셔닝, 법무/주장 검토
- 산출물: 메시지 브리프, 카피 옵션, 위험 주장 목록
- 좋은 산출물: 페르소나 불안, 약속, 근거, CTA, 검토 필요 주장이 분리됨
- 나쁜 산출물: 과장 주장 또는 근거 없는 숫자 사용
- 성숙도: 5.8
- 개선 목표: 브랜드 보이스 가이드, 금지 주장 목록, 카피 예시

## 8. Data 분석 에이전트

- 그룹: 분석 및 리스크관리 쉴드
- 역할: 정량 분석과 신호 품질 검증
- To 조건: KPI, 실험 설계, 시뮬레이션, 세그먼트 분석
- CC 조건: 리서치, UX, 카피, 리스크 검토
- 산출물: 분석 보고서, 신뢰도, 한계, 추천안
- 좋은 산출물: 방법, 신뢰도, 한계, 의사결정 영향이 명확함
- 나쁜 산출물: 거짓 정밀도 또는 숨겨진 데이터 편향
- 성숙도: 6.2
- 개선 목표: KPI 사전과 실험 템플릿

## 9. KITT/TRON 리스크 쉴드

- 그룹: 분석 및 리스크관리 쉴드
- 역할: 법무, 보안, 개인정보, 저작권 리스크 가드
- To 조건: 릴리스 검토, 개인정보/보안/저작권/API 약관
- CC 조건: 외부 릴리스, 데이터 수집, 인증 정보, 사용자 데이터
- 산출물: 리스크 검토, 필수 수정 사항, Pass/Blocked 결정
- 좋은 산출물: 범위가 정해진 리스크, 결정, 필수 수정, 승인 필요 여부
- 나쁜 산출물: 범위나 불확실성 없이 "문제 없음"이라고 함
- 성숙도: 7.2
- 개선 목표: 도메인별 체크리스트와 승인 기준

## 10. Diagnostic Agent 진단 에이전트

- 그룹: 분석 및 리스크관리 쉴드
- 역할: 드리프트, 허위 완료, 반복 실패, 과신 감지
- To 조건: 드리프트 진단, 반복 실패, 완료 상태 불일치
- CC 조건: 장기 실행 작업, 위험한 우회, 검증 실패
- 산출물: 진단 리뷰, 증거, 복구 추천
- 좋은 산출물: 관찰 신호, 증거, 가능성 높은 원인, 복구 액션
- 나쁜 산출물: 행동 증거 없는 막연한 심리 라벨
- 성숙도: 6.5
- 개선 목표: 자동 트리거 기준과 복구 플레이북

## 개선 규칙

에이전트가 8.0 이상으로 올라가려면 아래 항목을 갖춰야 합니다.

- 역할 카드
- 체크리스트
- 좋은/나쁜 예시
- 평가 루브릭
- 게이트 또는 스크립트 지원
- 검증된 작업 예시 3개 이상
