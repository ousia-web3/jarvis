# 에피소딕 메모리: 프로젝트 폴더 및 파일 관리 적정성 점검

## 상황

사용자는 Jarvis 프로젝트의 폴더와 파일 관리가 적절한지 점검해 달라고 요청했다. 최근 루트 README와 데이터/리서치 지침, 매뉴얼, Joi 표기 정리 등 문서가 빠르게 늘어난 상태였다.

## 관찰

프로젝트는 역할별 문서와 작업 요청, 기억 저장소, 대시보드, 스크립트가 잘 분리되어 있다. 반면 `.playwright-mcp/`, `tmp/`, 대형 PDF 추출 자료, 작업 요청 evidence가 장기적으로 누적될 수 있는 구조다.

## 판단

지금 당장 구조를 크게 바꿀 필요는 없다. 먼저 생성물과 원본 보존 자료의 정책을 문서화하고, Git 저장소 전환 가능성에 대비해 `.gitignore`를 두는 것이 비용 대비 효과가 가장 크다.

## 재사용할 교훈

운영 문서가 늘어날수록 “어디에 둘 것인가”보다 “원본, 살아있는 문서, 임시물, 증거, 아카이브를 어떻게 구분할 것인가”가 더 중요해진다.
