## 회고

- 오늘 내가 맡은 일: 실시간 작업 이벤트 로그의 문장형 상세 문구를 작업 항목형으로 정리했다.
- 실제로 한 일: 대시보드 표시 함수, 시작/완료 훅, 이벤트 스키마, 기존 JSONL 상세 문구를 함께 정리했다.
- 어려웠던 지점: 과거 로그에는 `했다`가 끝문장뿐 아니라 중간 문장에도 섞여 있어 화면 표시 레이어와 원본 JSONL을 함께 다뤄야 했다.
- 판단을 바꾼 순간: 단순히 예시 한 줄만 바꾸면 새 이벤트가 다시 문장형으로 들어올 수 있어 스크립트 정규화까지 포함했다.
- 인간 대표 또는 다른 에이전트에게 받은 피드백: 사용자는 완료 보고식 문장보다 대시보드에서 바로 스캔 가능한 작업 항목형 문구를 원했다.

- 새로 배운 점: 실시간 로그는 보고서 문장보다 짧은 상태 항목이 더 잘 읽힌다.
- 다음에 반복하지 말아야 할 실수: 대시보드에 노출되는 문구를 자연어 완료문으로 길게 남기지 않는다.
- 다음에도 재사용할 수 있는 패턴: 이벤트 원본, 표시 함수, 스키마 원칙, 생성 훅을 함께 맞춘다.
- 지혜 승격 후보: “실시간 이벤트 로그의 `detail`은 문장이 아니라 작업 항목으로 작성한다.”
