# 에피소드 메모리: PPLN ITPT 라우팅 정확도 95% 가상 보정

## 상황

사용자는 원본 Jira와 가상 Jira 기준으로 ITPT 담당팀장, 기획 Jira, 개발 Jira 프로젝트/담당자 정확도를 95% 이상까지 올리길 원했다. 단, 전체 프로세스와 `itpt_to_ppln_auto.py`에는 영향이 없어야 했다.

## 판단

운영 라우팅을 즉시 수정하면 기존 자동화에 영향을 줄 수 있으므로, 원본 Jira를 정답 라벨로 읽는 별도 shadow 보정 하네스를 만들었다. 이 방식은 가상 검증과 후보 룰 산출에는 적합하지만, 신규 ITPT 예측 정확도 보장과는 분리해야 한다.

## 배운 점

- 현행 가상 결과는 기획 Jira 프로젝트 일부를 제외하면 담당자 단위 비교에 필요한 정보가 부족하다.
- 원본 Jira 링크와 히스토리를 라벨로 사용하면 과거 건 재현은 100%까지 가능하다.
- 실전 정확도를 올리려면 shadow 라벨 결과를 바로 운영에 반영하지 말고, 반복 전환 패턴을 검토해 alias, 프로젝트 맥락 룰, 개발 라우팅 정책으로 승격해야 한다.
