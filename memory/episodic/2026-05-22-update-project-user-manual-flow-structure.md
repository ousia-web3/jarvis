# 에피소딕 메모리: 프로젝트 사용자 매뉴얼 흐름 구조 업데이트

## 회고

- 오늘 내가 맡은 일: 프로젝트 구성을 검토해 사용자 매뉴얼에 PUML 흐름도와 폴더 구조 설명을 반영했다.
- 실제로 한 일: 시작 문서, 오케스트레이션 스킬, 역할 인덱스를 확인하고 `project-user-manual.html`에 PlantUML 코드 블록과 프로젝트 폴더 구조 섹션을 추가했다.
- 어려웠던 지점: 폴더 역할을 표로 넣으면 데스크톱에서는 괜찮지만 모바일에서 좁게 접혀 읽기 불편했다.
- 판단을 바꾼 순간: 모바일 검증 스크린샷에서 표 가독성이 떨어지는 것을 보고 카드형 그리드로 바꿨다.
- 인간 대표 또는 다른 에이전트에게 받은 피드백: 사용자가 명시적으로 “프로젝트 구성 검토”와 “puml 형태 흐름도”를 요구했기 때문에 기존 설명보다 실제 구조와 재사용 가능한 소스 흐름도에 초점을 맞췄다.

## 학습

- 새로 배운 점: 문서 매뉴얼의 폴더 구조 설명은 표보다 카드형 탐색 단위가 모바일에서 안정적이다.
- 다음에 반복하지 말아야 할 실수: 데스크톱에서만 표가 좋아 보인다고 바로 완료하지 말고 모바일 렌더링을 확인해야 한다.
- 다음에도 재사용할 수 있는 패턴: 정적 문서 변경은 `rg`로 구조를 검토하고, Playwright로 주요 텍스트, 깨진 이미지, 콘솔 오류, 수평 오버플로우를 함께 확인한다.
- 지혜 승격 후보: “사용자 매뉴얼에 프로젝트 구조를 넣을 때는 실제 루트 스냅샷, 폴더별 역할, 탐색 순서를 한 묶음으로 제공한다.”
