# 에피소딕 메모리: 프로젝트 파일 관리 개선과 매뉴얼 업데이트

## 상황

파일 관리 감사에서 생성물, 임시 로그, 대형 PDF 추출 자료, 작업 요청 evidence, 폴더별 README, 과거 Joi 병기 표기가 개선 후보로 확인되었다. 사용자는 이 항목을 실제로 개선하고 매뉴얼까지 업데이트하라고 요청했다.

## 판단

파일 삭제나 이동은 링크 깨짐과 증거 손실 위험이 있으므로 먼저 비파괴 개선을 적용했다. 핵심은 `.gitignore`, 파일 관리 정책, 작업 요청 건강도 점검 스크립트, 폴더별 README를 통해 앞으로의 파일 흐름을 안정화하는 것이다.

## 결과

- 원본과 생성물의 경계가 문서화되었다.
- 브라우저 검증 산출물과 임시 로그는 추적 제외 기준에 들어갔다.
- 작업 요청 폴더 건강도를 점검하는 스크립트가 생겼다.
- 매뉴얼에서 파일 관리 유지보수와 문제 해결 흐름을 안내하게 되었다.

## 재사용할 교훈

운영체계형 프로젝트에서는 정리를 “삭제”로 시작하기보다 “분류 기준, 추적 제외, 건강도 점검, 매뉴얼 반영”으로 시작하는 편이 안전하다.
