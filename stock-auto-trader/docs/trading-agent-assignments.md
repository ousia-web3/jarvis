# 트레이딩 에이전트 배정

## 목적

목표 수익률 최소 25% 기준을 달성하기 위한 개발 방향을 에이전트별로 나눈다. 이 기준은 수익 보장이 아니라 전략 평가 기준선, 목표 도달 시 신규 주문 중단 조건, 리스크 리뷰 기준으로 사용한다.

## Human Brief 인간 브리프 업데이트

- 요청: 추후 토스 API를 활용해 종목 실시간 데이터, 데이터 관리 최적화, 증시 이슈 리서치를 반영한다.
- 안전한 해석: 실제 토스증권 API 문서와 권한이 확보되기 전까지는 placeholder와 인터페이스만 만든다.
- 완료 기준: 실시간 시세 품질 게이트, 데이터 저장소, 공식 리서치 소스 레지스트리, 이슈 브리프 엔진, 역할 분장이 문서와 코드에 반영된다.

## 데이터 관리 최적화 기준

- 실시간 스냅샷은 `timestamp`, `price`, `bid`, `ask`, `volume`, `source`, `latency_ms`를 가진다.
- 기본 freshness 기준은 5초이며, 기준을 넘으면 신규 주문 판단에 사용하지 않는다.
- bid/ask spread가 3%를 넘으면 이상치로 보고 차단한다.
- API 키, 계좌번호, 주문내역, 개인정보는 저장하지 않는다.
- 실거래 전에는 수수료, 세금, 환율, 슬리피지, 부분 체결, 휴장일, 데이터 지연을 반드시 모델링한다.

## 공식 리서치 소스

- 한국 공시: OpenDART
- 한국 시장 데이터: KRX Data Marketplace. 실시간 데이터는 라이선스가 필요하다.
- 미국 공시: SEC EDGAR APIs
- 토스증권: 공식 주문/실시간 API 문서가 확보되기 전까지 broker placeholder로 유지한다.

## 리스크 Shield 판정

- 실거래 주문, 토스 인증 정보, 계좌 접근, 외부 배포는 High 이상 리스크다.
- 현재 반영 범위는 로컬 인터페이스와 테스트이므로 Medium으로 진행 가능하다.
- 증시 이슈 리서치 결과는 투자 추천이 아니라 Risk Shield 입력 자료로만 사용한다.
