# PC/MO 반응형 웹 최적화 검토

## 1. 결론

해당 작업 폴더 내에서 PC 및 MO 최적화 작업은 가능하다. 다만 현재 폴더는 PRD, TRD, IA, 디자인 시스템, 태스크 문서 중심의 기획 산출물이며 실제 프론트엔드 구현 파일은 포함하지 않는다.

따라서 현재 단계에서 가능한 작업은 다음 두 갈래다.

| 경로 | 가능 여부 | 설명 |
| --- | --- | --- |
| 문서 보강 | 가능 | 브레이크포인트, 화면별 PC/MO 레이아웃, 검증 기준을 산출물로 추가 |
| 정적 목업 | 가능 | 작업 폴더 안에 `web/` 또는 `prototype/`을 만들고 반응형 HTML/CSS 목업 작성 |
| 실제 앱 구현 | 조건부 가능 | Next.js 등 구현 위치와 실행 환경을 먼저 정해야 함 |

권장 순서는 `문서 보강 -> 정적 반응형 목업 -> 실제 앱 구현`이다.

## 2. 현재 반응형 근거

이미 정의된 기준은 다음과 같다.

- `deliverables/02-trd.md`: `NFR-003`에 모바일 우선 반응형 요구사항이 있다.
- `deliverables/03-ia-brief.md`: 상세 공통 배너의 PC 16:9, 모바일 4:3 허용과 최소 높이가 정의되어 있다.
- `deliverables/06-design-system.md`: 모바일 탐색 우선, 데스크톱 좌측 필터/우측 결과 그리드, aspect-ratio 고정 원칙이 있다.
- `deliverables/08-coding-convention-ai-guide.md`: 모바일/데스크톱 배너 비율 유지, 긴 한국어 제목 2줄 처리, placeholder 상태 표시가 UI 테스트 기준으로 있다.

이 기준은 방향성으로는 충분하지만, 개발자가 바로 구현하기에는 화면별 상세 규격이 부족하다.

## 3. 보강이 필요한 항목

| 항목 | 현재 상태 | 필요한 보강 |
| --- | --- | --- |
| Breakpoint | 명시 부족 | 360, 390, 430, 768, 1024, 1280 기준 정의 |
| 홈 | IA만 있음 | PC hero/search/segment 배치와 MO 단일 컬럼 순서 정의 |
| 탐색 목록 | 원칙만 있음 | PC 좌측 필터 + 결과 그리드, MO 필터 바텀시트 정의 |
| 상세 페이지 | 배너 규칙 있음 | PC 본문/메타 2열, MO 정보 순서와 sticky CTA 정의 |
| 영상 영감 | 흐름만 있음 | 영상 카드와 원본 링크 CTA의 PC/MO 배치 정의 |
| 코스 | 흐름만 있음 | 코스 트레이, 저장함, 공유 버튼의 모바일 고정 액션 정의 |
| Admin/CMS | 메뉴 구조 있음 | PC 업무형 화면 우선, MO는 조회/승인 최소 플로우로 축소 |
| 검증 | 테스트 항목 일부 있음 | viewport별 스크린샷, 가로 스크롤 금지, 터치 타깃 기준 추가 |

## 4. 권장 브레이크포인트

| 구간 | 너비 | 레이아웃 기준 |
| --- | --- | --- |
| MO-S | 360px | 최소 지원 모바일, 단일 컬럼, CTA 1개 우선 |
| MO-M | 390px | 기본 모바일 기준, 필터 바텀시트 |
| MO-L | 430px | 큰 모바일, 카드 여백과 태그 줄바꿈 검증 |
| Tablet | 768px | 2열 카드 또는 필터 drawer |
| PC | 1024px | 좌측 필터 + 우측 결과, 3열 카드 |
| Wide PC | 1280px 이상 | 최대 폭 1200~1280px, 4열 카드 허용 |

모든 화면은 `360px`에서 가로 스크롤이 없어야 한다.

## 5. 화면별 최적화 방향

### 5.1 공개 웹 홈

- PC: 상단 검색, 국적/권역 선택, 동행 유형 선택, 추천 콘텐츠 그리드를 한 화면 안에서 스캔 가능하게 배치한다.
- MO: 검색 -> 국적/권역 -> 동행 유형 -> 추천 카드 순서로 단일 컬럼을 둔다.
- MO 하단 고정 영역에는 `필터`, `저장함`처럼 반복 행동만 둔다.

### 5.2 탐색 목록

- PC: 좌측 `FilterPanel`을 고정 폭으로 두고, 우측에 3~4열 결과 그리드를 둔다.
- Tablet: 필터는 drawer 또는 상단 collapse로 전환한다.
- MO: 필터는 바텀시트로 열고, 결과 카드는 1열로 둔다.
- 카드 썸네일은 4:3 aspect-ratio를 고정해 이미지 교체 시 레이아웃 밀림을 막는다.

### 5.3 상세 페이지

- PC: 상단 `VisualBannerPlaceholder` 아래에 제목/태그/요약을 두고, 본문과 기본 정보를 2열로 분리한다.
- MO: 배너 -> 제목 -> 핵심 태그 -> 추천 이유 -> 기본 정보 -> 관련 콘텐츠 순서로 둔다.
- `코스에 담기` 같은 주요 CTA는 모바일에서 하단 sticky action으로 둘 수 있다.
- 배너는 PC 16:9 최소 320px, MO 4:3 또는 16:9 최소 180px을 유지한다.

### 5.4 영상 영감

- 원본 영상은 재호스팅하지 않고 링크 CTA만 제공한다.
- PC: 영상 영감 카드와 연결 여행정보를 나란히 비교할 수 있게 한다.
- MO: 영상 카드, 추출 태그, 연결된 지역/음식/관광지를 순차 카드로 보여준다.
- 유튜브 썸네일을 직접 저장/재배포하지 않는 정책을 UI 상태 텍스트로 오해 없이 표시한다.

### 5.5 Trip Ontology Dashboard

- 운영 대시보드는 PC 업무 화면을 기본으로 한다.
- MO에서는 전체 그래프 편집보다 Overview, Relation Review 승인/보류, Risk 요약 조회를 우선한다.
- 그래프 캔버스는 모바일에서 목록/관계 테이블 대체 뷰를 제공하는 것이 안전하다.

## 6. UI 검증 기준

| 기준 | Pass 조건 |
| --- | --- |
| Viewport | 360, 390, 430, 768, 1024, 1280px에서 주요 화면 확인 |
| 가로 스크롤 | 모든 공개 웹 화면에서 없어야 함 |
| 터치 타깃 | 모바일 주요 버튼과 필터 칩은 최소 44px 높이 |
| 텍스트 | 긴 한국어/영어/일본어 라벨은 2줄 처리 또는 말줄임 |
| 이미지 슬롯 | placeholder, licensed, owned, public_license, blocked 상태가 레이아웃을 흔들지 않음 |
| 필터 | MO에서 바텀시트/드로어가 콘텐츠를 가리지 않고 닫기 가능 |
| 접근성 | 이미지 슬롯 대체 텍스트, 상태 텍스트, 키보드 포커스 확인 |
