# Design System

## 1. 디자인 방향

- 톤: 신뢰감 있는 여행 큐레이션, 따뜻하지만 과장 없는 설명.
- 화면 성격: 여행 콘텐츠 서비스이지만 초기에는 실제 이미지가 없으므로 정보 구조와 플레이스홀더 완성도가 중요하다.
- 핵심 원칙:
  - 지역, 음식, 관광지를 빠르게 비교할 수 있어야 한다.
  - 모든 시각 영역은 동일한 규격의 공통 배너로 시작한다.
  - 국적/동행 추천은 설명 가능해야 한다.
  - 모바일 탐색을 우선한다.

## 2. 공통 배너 컴포넌트

### 컴포넌트명

`VisualBannerPlaceholder`

### 용도

지역, 음식, 관광지, 코스, 영상 영감 상세 상단의 시각 정보 영역.

### 구조

```text
┌──────────────────────────────────────────────┐
│ 이미지 준비중                                │
│ [콘텐츠 유형] 대표 이미지가 들어갈 영역입니다 │
│ alt: 하동 녹차밭 전경 이미지 예정            │
└──────────────────────────────────────────────┘
```

### 상태

| 상태 | 표시 |
| --- | --- |
| placeholder | 이미지 준비중 |
| licensed | 공식/라이선스 이미지 |
| owned | 직접 촬영 이미지 |
| public_license | 공공 라이선스 이미지 |
| blocked | 권리 확인 불가 |

### 크기

- 데스크톱: 16:9, 최소 높이 320px
- 태블릿: 16:9, 최소 높이 260px
- 모바일: 4:3 또는 16:9, 최소 높이 180px
- 카드 썸네일: 4:3, 고정 aspect-ratio

### 접근성

- 대체 텍스트 필수
- `placeholder`는 장식 이미지가 아니라 정보 상태이므로 상태 텍스트를 제공
- 색상만으로 상태를 구분하지 않음

## 3. 주요 컴포넌트

| 컴포넌트 | 용도 |
| --- | --- |
| SegmentSelector | 국적/권역, 동행 유형 선택 |

| TravelDifficultyMeter | 이동/음식/체험 난이도 |

## 4. 콘텐츠 카드 정보 순서

1. 공통 이미지 영역 또는 썸네일 플레이스홀더
2. 콘텐츠 유형 배지
3. 제목
4. 지역 또는 메뉴
5. 추천 대상: 국적/동행/관심사
6. 추천 이유 1줄
7. 코스에 담기 버튼

## 5. 상세 페이지 정보 순서

1. 공통 배너 이미지 영역
2. 제목과 핵심 태그
3. 한 줄 요약
4. 추천 대상과 추천 이유
5. 기본 정보: 위치, 소요 시간, 이동 난이도, 운영시간, 비용
6. 음식/관광/영상 연결
7. 관련 코스
8. 출처와 검수일

## 6. 라벨 톤

- 권장:
  - `이런 여행자에게 좋아요`
  - `영상에서 발견한 포인트`
  - `처음 가도 쉬운 이유`
  - `주의하면 좋은 점`
- 금지:
  - `이 국적은 무조건 좋아함`
  - `최고`, `필수`, `완벽` 같은 과장형 단정
  - 권리 확인 전 `공식 이미지` 표기

## 7. 색상과 시각 스타일 초안

실제 UI 구현 전 가이드이며, 한 가지 색상에 과도하게 의존하지 않는다.

| 역할 | 방향 |
| --- | --- |
| Base | 흰색 또는 매우 밝은 회색 |
| Text | 높은 대비의 차콜 |
| Primary | 한국 여행/지도 감각의 청록 계열 |
| Accent | 음식/경험 강조용 코랄 계열 |
| Support | 자연/지역 강조용 녹색 계열 |
| Warning | 권리/검수 상태용 노랑/주황 |

## 8. 반응형 원칙

- 모바일 하단에는 필터와 저장함 접근을 고정한다.
- 데스크톱은 좌측 필터, 우측 결과 그리드 구조를 기본으로 한다.
- 배너와 카드의 aspect-ratio를 고정해 이미지 교체 시 레이아웃 밀림을 방지한다.
- 긴 한국어 메뉴명은 2줄까지 허용하고 이후 말줄임 처리한다.

## 9. Trip Ontology Dashboard 컴포넌트

| 컴포넌트 | 용도 | 시각 원칙 |
| --- | --- | --- |
| OntologyHealthSummary | 노드/관계/근거/권리 상태 요약 | 숫자 카드 4개 이하, 위험 상태는 색상과 텍스트 병기 |
| OntologyGraphCanvas | 지역, 음식, 관광지, 영상, 세그먼트 관계 탐색 | 노드 색상은 유형별로 구분하되 한 색상 계열에 의존하지 않음 |
| RelationReviewQueue | 승인 대기 관계 후보 검수 | 근거, 출처, 권리 상태, 영향 콘텐츠를 한 행에서 확인 |
| CoverageGapTable | 고아 노드와 커버리지 공백 확인 | 우선순위, 담당자, 마지막 검수일을 고정 열로 제공 |
| RiskImpactPanel | 권리/출처 문제가 연결된 콘텐츠 영향 범위 | 공개 차단, 보류, 수정 요청 상태를 명확히 분리 |
| QueryLabResult | 운영자 그래프 질의 결과 | 질의 결과와 사용 가능 여부를 분리 표시 |

Dashboard 화면은 운영 도구이므로 장식적 그래픽보다 빠른 스캔과 검수 효율을 우선한다. 그래프 영역은 전체 화면의 핵심 작업 공간으로 두고, 검수 패널은 우측 또는 하단 분할 영역에 배치한다.
