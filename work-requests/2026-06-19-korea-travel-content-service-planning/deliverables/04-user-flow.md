# User Flow

## 1. 외국인 여행자: 국적과 동행 기반 탐색

```mermaid
flowchart TD
  A[홈] --> B[국적/권역 선택]
  B --> C[동행 유형 선택]
  C --> D[추천 콘텐츠 목록]
  D --> E[지역/음식/관광지 상세]
  E --> F[코스에 담기]
  F --> G[저장함 또는 공유]
```

성공 기준:
- 사용자는 국적과 동행 유형을 선택한 뒤 2클릭 이내 상세 페이지에 도달한다.
- 상세 페이지에는 추천 이유와 공통 배너 이미지 영역이 표시된다.

## 2. 유튜브 콘텐츠에서 여행정보로 이동

```mermaid
flowchart TD
  A[영상 영감 목록] --> B[에피소드/클립 카드]
  B --> C[추출 태그 확인]
  C --> D{연결 정보}
  D --> E[지역 상세]
  D --> F[음식 상세]
  D --> G[관광지 상세]
  E --> H[관련 코스]
  F --> H
  G --> H
```

성공 기준:
- 영상 카드는 원본 유튜브 링크와 분석 태그를 제공한다.
- 영상 이미지를 직접 재사용하지 않고 플레이스홀더를 사용한다.

## 3. 한국 거주 초대자: 외국인 친구 코스 제안

```mermaid
flowchart TD
  A[홈] --> B[동행별 여행]
  B --> C[친구/가족/부모 동반 선택]
  C --> D[관심사 선택: 음식/자연/한류/역사]
  D --> E[코스 후보]
  E --> F[이동 난이도 확인]
  F --> G[공유 링크 생성]
```

성공 기준:
- 코스는 이동 난이도, 식사 난이도, 소요 시간을 함께 보여준다.
- 부모 동반/아이 동반은 휴식 지점과 접근성 정보를 강조한다.

## 4. 음식 중심 탐색

```mermaid
flowchart TD
  A[탐색] --> B[음식]
  B --> C[업종 선택]
  C --> D[메뉴 선택]
  D --> E[맵기/재료/식사상황 필터]
  E --> F[음식 상세]
  F --> G[이 음식을 먹을 수 있는 지역/코스]
```

성공 기준:
- 메뉴 상세는 `맛`, `먹는 방법`, `주의 재료`, `추천 동행`, `영상 영감`을 포함한다.

## 5. 운영자 콘텐츠 등록

```mermaid
flowchart TD
  A[Admin 로그인] --> B[영상 URL 등록]
  B --> C[메타데이터 수집]
  C --> D[태그 입력]
  D --> E[지역/음식/관광지 연결]
  E --> F[이미지 슬롯 상태 지정]
  F --> G[권리/출처 검수]
  G --> H{승인}
  H -->|Pass| I[공개]
  H -->|Needs Work| J[수정 큐]
```

성공 기준:
- 공개 전 권리 상태와 검수 상태가 필수다.
- 이미지가 없으면 자동으로 공통 배너 플레이스홀더가 노출된다.

## 6. 다국어 확장

```mermaid
flowchart TD
  A[한국어 원문 콘텐츠] --> B[번역 요청]
  B --> C[영문/일문 초벌]
  C --> D[현지화 검수]
  D --> E[언어별 공개]
  E --> F[언어별 성과 분석]
```

성공 기준:
- 번역 콘텐츠는 원문 콘텐츠와 같은 ID 체계를 공유한다.
- 언어별 공개 상태를 독립적으로 관리한다.

## 7. 운영자 Trip Ontology 검수

```mermaid
flowchart TD
  A[Ontology Dashboard] --> B[용어 사전 확인]
  B --> C[관계 후보 큐]
  C --> D{근거와 권리 상태}
  D -->|충분| E[관계 승인]
  D -->|부족| F[보류 또는 반려]
  E --> G[Graph Projection 갱신]
  G --> H[검색/추천/상세 근거 반영]
  F --> I[수정 작업 큐]
```

성공 기준:
- 관계 후보는 `Evidence`, 출처, 권리 상태를 함께 확인한 뒤 승인한다.
- 승인되지 않은 관계는 공개 추천 이유나 사용자 화면의 단정 근거로 사용하지 않는다.
- 고아 노드, 권리 영향 콘텐츠, 커버리지 공백은 운영자 대시보드에서 우선 작업 큐로 전환된다.
