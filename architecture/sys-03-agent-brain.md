# SYS.03 The Agent Brain: 기억과 망각의 퓨전 아키텍처

## 목적

에이전트 팀이 단순히 많은 문서를 기억하는 것이 아니라, 실제 업무 수행에서 얻은 경험을 지혜로 승격하고 불필요한 컨텍스트를 망각하도록 설계합니다.

## 기억 계층

| 계층 | 설명 | 보존 정책 |
| --- | --- | --- |
| 작업 메모리 | 현재 태스크 수행에 필요한 단기 컨텍스트 | 태스크 종료 후 요약 |
| 에피소딕 메모리 | 언제, 누가, 왜, 무엇을 했는지 기록한 일기 | 스프린트 단위 보존 |
| 의미 메모리 | 검증된 규칙, 체크리스트, 반복 가능한 절차 | 장기 보존 |
| 지혜 저장소 | 여러 경험에서 추출된 공통 원리 | 승인 후 영구 보존 |
| 망각 큐 | 오래된 로우 데이터, 중복 로그, 오류 유발 컨텍스트 | 주기적 소거 |

## 지혜 승격

1. 에이전트가 업무 로그와 일기를 남긴다.
2. Data가 반복 패턴과 실패 패턴을 식별한다.
3. Jarvis가 공통 원리 후보를 검토한다.
4. Friday가 운영 규칙 또는 체크리스트로 반영한다.
5. 인간 대표가 필요 시 최종 승인한다.

## 망각 정책

- 핵심 지혜와 승인된 운영 규칙은 보존한다.
- 로우 데이터와 중복 로그는 요약 후 소거한다.
- 개인정보와 민감정보는 최소한으로 보존한다.
- 에이전트 판단을 오염시키는 실패 컨텍스트는 격리한다.
- 드리프트 사례는 요약본과 대응 원칙만 남긴다.

## 완료 기준

- 모든 완료 태스크는 회고 또는 로그를 남긴다.
- 반복 패턴은 지혜 후보로 등록된다.
- 불필요한 컨텍스트는 정책에 따라 제거된다.
