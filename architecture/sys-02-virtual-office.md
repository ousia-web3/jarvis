# SYS.02 The Virtual Office: 자율 통신망

## 목적

에이전트가 고립된 채팅창에서 따로 움직이지 않고, 하나의 가상 사무실에서 실시간으로 협업하도록 합니다. Discord 또는 동등한 채널형 협업 도구를 전제로 하며, 모든 업무 흐름은 To/CC 기반 메시지로 추적됩니다.

## 채널 구조

| 채널 | 목적 | 주 담당 |
| --- | --- | --- |
| `#command-bridge` | 인간 대표, Jarvis, Friday의 지휘 채널 | Jarvis |
| `#prj-{project-name}` | 프로젝트별 실행 채널 | Friday |
| `#agent-{name}` | 에이전트별 사고 과정과 산출물 로그 | 각 에이전트 |
| `#risk-shield` | 법무·보안·드리프트 리스크 검토 | KITT/TRON |
| `#memory-log` | 에피소딕 메모리와 회고 기록 | Data |
| `#general-logs` | 시스템 이벤트와 자동화 로그 | Friday |

## To/CC 규칙

- `To`: 풀 컨텍스트를 받는 실행 책임자입니다.
- `CC`: 요약 컨텍스트를 받는 검토·감시자입니다.
- `Escalate`: 상위 판단이 필요한 경우 Jarvis 또는 인간 대표에게 올립니다.
- KITT/TRON은 리스크가 있는 작업에 강제 CC로 추가됩니다.
- Data는 지표, 실험, 시뮬레이션, 사용자 분석 작업에 강제 CC로 추가됩니다.

## 메시지 생명주기

1. Friday가 태스크를 분해하고 To/CC를 지정한다.
2. Owner가 실행 로그와 산출물을 남긴다.
3. CC가 필요한 범위만 검토한다.
4. 리스크 발생 시 `#risk-shield`로 복사하고 승격한다.
5. 완료 후 `#memory-log`에 회고를 남긴다.

## 완료 기준

- 채널 목적과 책임자가 명확하다.
- 풀 컨텍스트와 요약 컨텍스트가 분리된다.
- 인간 대표가 채널 피드만 보고도 진행 상황을 감사할 수 있다.
