# SYS.01 The Dream Team: 역할 기반 에이전트 조직

## 목적

전능한 단일 AI가 아니라, 전문성·권한·성격·보고 라인을 가진 에이전트 조직을 구성합니다. 이 단계의 핵심은 "무엇을 할 것인가"보다 "누가 어떤 책임으로 일할 것인가"를 먼저 확정하는 것입니다.

## 계층 구조

| 계층 | 구성원 | 핵심 책임 |
| --- | --- | --- |
| 지휘부 | 인간 대표, Jarvis | 비전, 전략, 최종 결정, 시스템 방향성 |
| 프로젝트 매니저 | Friday | 업무 분해, To/CC 지정, 일정·리스크·산출물 취합 |
| 기획 및 실무진 | EVE, Joi, TARS, C3PO | 리서치, UX/UI, 개발, 마케팅 카피 실행 |
| 분석 및 리스크관리 쉴드 | Data, KITT/TRON, 진단 에이전트 | 데이터 검증, 법무·보안, 드리프트 감시 |

## 운영 원칙

- 모든 에이전트는 고유한 `mission`, `authority`, `inputs`, `outputs`, `risk_flags`를 가진다.
- 모든 태스크는 `Owner(To)`와 `CC`를 반드시 갖는다.
- 실무진은 직접 실행하지만, 외부 공개·개인정보·법무·보안 이슈는 KITT/TRON 검토를 통과해야 한다.
- Data는 정량 주장과 실험 결과의 근거를 확인한다.
- Jarvis는 전략 충돌과 인간 승인 필요 사안을 판단한다.

- 태스크 분해 템플릿: `../templates/friday-task-breakdown-template.md`

## 완료 기준

- 지휘부, PM, 실무진, 쉴드가 모두 지정되어 있다.
- 모든 에이전트의 권한과 금지사항이 분리되어 있다.
- 태스크 배정 시 책임자와 검토자가 혼동되지 않는다.
